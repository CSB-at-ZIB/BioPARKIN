Detailed Description

The network controller which basically manages all aspects revolving around a single SBML model (i.e.

SBML file). It has references to outside Views (e.g. TreeView). Internally, it has a reference to the custom NetworkView. A Dirty state is used to handle the need for saving/rejecting changes, etc.

It inherits from QObject for the use of Signals/Slots.

Parameters:
filename,:Filename of the SBML model that should be opened. If None, an empty model is created. filename: str
views,:The views of the main window that are to be associated with the model. views: L{QAbstractItemView}
